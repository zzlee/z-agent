# Web 界面設計

## 1. 設計目標

- **降低複製貼上摩擦**：最小化使用者在系統與外部 LLM 之間切換的操作成本
- **清晰的狀態呈現**：一眼看出 Agent 目前在哪個階段
- **工具呼叫透明度**：使用者完全理解和控制每個工具呼叫
- **響應式設計**：支援桌面和平板尺寸

### 2.2 對話歷史區域

在主要工作區域上方，展示完整的對話歷史，以類似聊天的方式呈現：

- 👤 **使用者訊息**: 藍色氣泡
- 🤖 **LLM 回應**: 灰色氣泡，內含 Markdown 渲染
- 🔧 **工具呼叫**: 黃色卡片，顯示工具名稱和參數
- 📄 **工具結果**: 綠色/紅色卡片，顯示成功/失敗結果

## 3. 核心 UI 元件

### 3.1 PromptDisplay（提示詞展示）
- 語法高亮顯示完整提示詞
- 可折疊/展開各區塊（System / History / Current）
- 一鍵複製功能
- 顯示估算 token 數
- 選項：複製完整提示詞 vs 僅最新部分

### 3.2 ResponseInput（回應輸入）
- 大型文字輸入區域（支援多行）
- 貼上時自動偵測格式
- 提供「解析預覽」功能：貼上後先顯示解析結果
- 支援拖放文字檔

### 3.3 ToolCallCard（工具呼叫與執行計畫組件）
- **階段分組展示**：依據依賴關係分析產生的 Stage 將工具呼叫分組顯示
- **並行/順序標記**：清晰指示哪些工具可同時執行，哪些有先後依賴關係
- **互動操作**：提供各工具參數的修改、刪除、或調整執行順序
- 執行中顯示個別工具的進度條與日誌輸出
- 完成後顯示各工具結果摘要與時程分析

### 3.4 SessionSidebar（會話側邊欄）
- 會話列表，依時間排序
- 顯示會話名稱、狀態、最後更新時間
- 搜尋和過濾功能
- 右鍵選單：重命名、刪除、匯出、分支

### 3.5 StatusBar（狀態列）
- 當前會話名稱
- Agent 狀態（圖示 + 文字）
- Token 估算和使用統計
- 工作目錄路徑

## 4. 操作流程 UX

### 4.1 新建任務流程
1. 使用者點擊「新會話」
2. 輸入任務描述
3. 系統自動組裝提示詞並顯示在提示詞區域
4. 使用者點擊「複製提示詞」
5. 使用者切換到外部 LLM，貼上提示詞
6. LLM 回應後，使用者複製回應
7. 使用者回到 Z-Agent，貼到回應輸入區域
8. 點擊「送出回應」
9. 系統解析並顯示工具呼叫（若有）
10. 使用者確認執行
11. 系統執行工具，組裝新提示詞
12. 重複步驟 4-11

### 4.2 快捷鍵

| 快捷鍵 | 功能 |
|--------|------|
| `Ctrl+Shift+C` | 複製提示詞 |
| `Ctrl+Enter` | 送出 LLM 回應 |
| `Ctrl+Shift+E` | 確認執行所有工具 |
| `Ctrl+N` | 新建會話 |
| `Escape` | 取消當前操作 |

## 5. 進階功能

### 5.1 提示詞模板
- 使用者可自訂系統提示詞模板
- 提供預設模板：通用助理、程式碼助理、文件助理
- 模板變數替換（{cwd}、{date}、{tools}）

### 5.2 多模型支援提示
- 針對不同 LLM 最佳化提示詞格式
- 可選目標模型：ChatGPT / Claude / Gemini / 其他
- 不同模型的工具呼叫格式可能略有差異

### 5.3 會話匯出
- 匯出為 Markdown
- 匯出為 JSON
- 匯出為 HTML（完整對話渲染）
