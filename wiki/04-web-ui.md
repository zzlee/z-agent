# Web 轉發控制台

## 1. 設計目標

- **最小化複製貼上摩擦**：讓使用者在 Z-Agent 與外部 LLM 之間快速切換
- **清晰的轉發狀態**：一眼看出目前的轉發階段（等待 LLM / 等待確認 / 執行中）
- **工具呼叫透明度**：使用者可清楚看到並控制每個即將執行的工具呼叫
- **響應式設計**：支援桌面與平板

## 2. 頁面佈局

### 2.2 對話歷史（可選展開）

在提示詞區域下方或側邊以簡潔的列表形式顯示最近的轉發記錄，讓使用者能追蹤目前為止的交互。

## 3. UI 元件

### 3.1 PromptDisplay（提示詞展示）
- 顯示組裝好的完整提示詞
- 可切換：完整提示詞 / 僅最新變更
- 一鍵複製按鈕
- 顯示估算 token 數

### 3.2 ResponseInput（回應輸入）
- 大型多行文字區
- 貼上後自動預覽解析結果

### 3.3 ToolPlanCard（工具執行計畫）
- 依 Stage 分組顯示工具呼叫
- 同一 Stage 標示「並行」，不同 Stage 標示「依序」
- 執行中顯示個別工具的進度狀態
- 確認執行按鈕

### 3.4 ResultCard（工具結果）
- 顯示執行完的工具結果
- 一鍵複製回 LLM

### 3.5 StatusBar（狀態列）
- 伺服器連線狀態
- 當前轉發階段
- Token 估算

## 4. 操作流程

1. 建立會話 → 輸入任務描述
2. 系統組裝提示詞 → 使用者點擊「複製提示詞」
3. 切換到外部 LLM 貼上提示詞 → LLM 回應
4. 複製 LLM 回應 → 貼回 Z-Agent → 點擊「提交並解析」
5. 檢視解析出的工具呼叫計畫 → 點擊「執行確認計畫」
6. 系統執行工具 → 顯示結果 → 使用者複製結果回 LLM
7. 重複步驟 2-6
