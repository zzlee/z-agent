# 實施計畫

## 實施概要

Z-Agent 作為 LLM Tool-Use 轉發閘道，實施重點在於**核心轉發循環的完整性**與**使用者體驗的流暢性**，而非功能的豐富度。

---

## Phase 1：核心轉發引擎（已完成）

### 目標
建立最小可用的轉發引擎，能夠組裝提示詞、解析 LLM 回應、分析依賴關係並執行工具。

### 交付模組

| 模組 | 說明 |
|------|------|
| 工具定義 (4 個) | read, write, edit, bash（搜尋透過 bash 呼叫 rg/grep） |
| PromptAssembler | 提示詞組裝器 |
| ResponseParser | LLM 回應解析器 |
| DependencyPlanner | DAG 依賴分析與執行計畫生成 |
| AgentEngine | 核心引擎整合 |

### 驗收標準
- [x] 五個工具都能獨立執行和回傳結果
- [x] 提示詞組裝器能正確生成包含工具描述的提示詞
- [x] 回應解析器能從文字中提取 tool_call 指令
- [x] 依賴分析模組能正確識別衝突並生成 Stage 執行計畫

---

## Phase 2：Web 轉發控制台（進行中）

### 目標
建立功能完整的 Web 前端介面，專注於轉發循環的核心體驗。

### 交付物

| 檔案 | 說明 |
|------|------|
| `web/index.html` | 主頁面（三區塊佈局） |
| `web/css/main.css` | 樣式 |
| `web/js/app.js` | 應用邏輯（純轉發循環） |
| `web/js/api-client.js` | API 封裝 |

### 核心功能
1. **提示詞展示與複製**：顯示組裝的提示詞、一鍵複製
2. **回應輸入與解析**：貼上 LLM 回應、提取 tool_call
3. **工具執行計畫與執行**：DAG 分組顯示、確認執行、即時進度
4. **結果展示與複製**：顯示工具執行結果、一鍵複製回 LLM
5. **輕量會話管理**：建立、切換、刪除會話

### 非目標（明確不包含）
- 會話分支
- 歷史壓縮
- 多模型提示詞適配
- 快捷鍵系統（可後續追加）
- 深色模式切換（預設深色主題）

---

## Phase 3：API 與後端服務（已完成）

### 交付物

| 端點 | 說明 |
|------|------|
| `GET/POST /api/sessions` | 會話 CRUD |
| `GET /api/sessions/:id/prompt` | 取得提示詞 |
| `POST /api/sessions/:id/response` | 提交 LLM 回應 |
| `POST /api/sessions/:id/execute` | 執行工具計畫 |
| `WebSocket /ws` | 執行進度推送 |

---

## 技術選型

| 類別 | 選擇 | 理由 |
|------|------|------|
| 後端 | Node.js + Express | 輕量、生態豐富 |
| 前端 | Vanilla JS + CSS | 零依賴、快速載入 |
| 儲存 | JSON/JSONL | 無需資料庫 |
| 即時通訊 | WebSocket | 原生支援 |

---

## 風險與緩解

| 風險 | 影響 | 緩解措施 |
|------|------|----------|
| LLM 回應格式不一致 | 解析失敗 | 容錯解析 + 手動編輯 |
| 複製貼上操作繁瑣 | 體驗差 | 一鍵複製 + 清晰指引 |
| 工具執行安全風險 | 系統損壞 | 路徑限制 + 命令黑名單 + 確認機制 |
